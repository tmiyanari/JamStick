# 渋滞ものさし - 詳細技術仕様書

## 1. 概要
本ツールは、Webブラウザのみで完結するサーバーレスの地図計測アプリケーションです。React と Leaflet をベースに、正確な物理距離に基づく目盛り描画と、高精度な画像エクスポート機能を提供します。

## 2. 技術スタック
-   **フレームワーク**: React 18 (Babel Standalone)
-   **スタイリング**: Tailwind CSS, CSS3 (Flexbox, Grid)
-   **地図ライブラリ**: Leaflet 1.9.4
-   **空間解析**: Turf.js 6.x (距離計算、補完点生成、方位角計算)
-   **ジオコーディング**: Nominatim (OpenStreetMap) API
-   **画像処理**: HTML5 Canvas API (カスタムキャプチャロジック)

## 3. 主要機能の実装詳細

### 3.1 距離計測と目盛り描画
-   **入力**: 地図クリックにより緯度経度（LatLng）の配列を取得。
-   **パス生成**: Turf.js を使用して LatLng 配列を GeoJSON LineString に変換。
-   **補完点生成**: `turf.along` を用い、起点から指定した間隔（デフォルト10m）ごとに補完点を生成。
-   **視覚効果**:
    -   **目盛り（Tick）**: 生成した各補完点において、パスの進行方向（Bearing）に対し直交する 10m〜30m のポリラインを描画。
    -   **距離ラベル**: 100m ごとの地点に `L.divIcon` を配置。パスの角度に合わせて CSS transform で回転。

### 3.2 A4 比率レイアウト
-   **レスポンシブ計算**: `CSS min()` 関数を使用し、画面サイズを超えない範囲で A4 用紙比率（1:1.414）を維持。
    -   `width: min(96vw, calc(96vh * 210 / 297))`
    -   `height: min(96vh, calc(96vw * 297 / 210))`
-   **不具合対策**: Leaflet はコンテナサイズの動的変更時にタイルが消失しやすいため、`.map-wrapper` を介した構造を採用し、サイズ確定後に `map.invalidateSize()` を実行。

### 3.3 スクリーンショット機能 (Canvas 合成)
-   **原理**: `html2canvas` などの汎用ライブラリを使用せず、Canvas API で直接描画。
-   **タイル描画**: `getBoundingClientRect()` で各タイルの実座標を取得し、`drawImage` で転写。
    -   **格子線対策**: 浮動小数点による端数（サブピクセル）を `Math.round()` で整数化し、タイル間の隙間を排除。
-   **オーバーレイ描画**: 地図上のパス、目盛り、ラベルを `map.latLngToContainerPoint()` でピクセル座標に変換し、Canvas 上に直接再描画。
-   **CORS対応**: タイル画像を `crossOrigin = 'anonymous'` で再取得し、汚染（Taint）を回避して `toDataURL` を実行可能に。

### 3.4 場所検索
-   Nominatim API を非同期フェッチし、結果をリスト表示。選択時に `map.setView()` で遷移。

## 4. 制限事項・注意事項
-   **タイル制限**: Google Maps レイヤーは Google の利用規約および技術的な CORS 制限により、スクリーンショットに含まれない場合があります。
-   **精度**: Turf.js は地球を楕円体として計算（WGS84）していますが、地図投影法（メルカトル）との誤差が極大ズーム時には生じる可能性があります。
